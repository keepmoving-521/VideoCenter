# R18-R19：解析超时与失败重试

资源解析器注册器现在统一负责超时控制和临时失败重试，具体网站解析器无需重复实现。

## 默认策略

| 配置 | 默认值 | 说明 |
| --- | --- | --- |
| `VIDEOCENTER_PARSER_TIMEOUT_SECONDS` | `30` | 每次解析尝试的最大秒数 |
| `VIDEOCENTER_PARSER_MAX_ATTEMPTS` | `3` | 包含首次调用在内的最大尝试次数 |
| `VIDEOCENTER_PARSER_RETRY_DELAY_SECONDS` | `0.5` | 第一次重试前的等待秒数 |
| `VIDEOCENTER_PARSER_RETRY_MAX_DELAY_SECONDS` | `5` | 单次退避等待上限 |

重试使用指数退避。例如默认配置下，两次重试前分别等待 0.5 秒和 1 秒。

## 可重试错误

以下临时错误会重试：

- 单次解析超时；
- 连接失败；
- `URLError` 等网络访问错误；
- HTTP 429；
- HTTP 5xx。

以下错误不会重试：

- URL 或解析结果参数校验失败；
- 页面数据结构不合法；
- 不支持的网站；
- 解析器返回了错误的数据类型；
- HTTP 4xx（429 除外）。

## API 错误

超过全部尝试次数后使用统一错误格式：

| HTTP 状态 | 错误代码 | 说明 |
| --- | --- | --- |
| 504 | `PARSE_TIMEOUT` | 每次尝试都超过解析时限 |
| 502 | `PARSE_FAILED` | 可重试错误在所有尝试中均未恢复 |

错误详情包含解析器名称、目标域名和尝试次数，但不会返回完整 URL，避免泄露查询参数中的令牌。

## 注意事项

超时按“每次尝试”计算，而不是整个请求的总时长。最大总时长还包括重试退避等待。生产环境可根据网络质量降低尝试次数或调整超时。

本次迭代不修改数据库结构，不需要新增 Alembic 迁移。
