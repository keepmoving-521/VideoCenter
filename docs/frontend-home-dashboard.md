# W04：实现首页数据概览

首页已从静态占位内容升级为真实数据概览，通过 W03 的统一 API 客户端读取后端数据。

## 数据来源

| 首页区域 | API |
| --- | --- |
| 影视条目、收藏、本地视频 | `GET /api/v1/media/stats` |
| 进行中下载 | `GET /api/v1/downloads` |
| 本周观看 | `GET /api/v1/history/stats/daily` |
| 继续观看 | `GET /api/v1/history/continue-watching` |
| 任务动态 | `GET /api/v1/tasks` |

首页加载时会并行请求以上接口，减少等待时间。

## 数据概览

首页概览卡片展示：

- 影视条目总数和收藏数量；
- 等待、下载中、暂停状态的下载任务数量；
- 本地视频数量和已有本地资源的影视数量；
- 从本周一到当前日期的累计观看时长和观看次数。

## 继续观看

最多读取三条未看完记录，展示：

- 影片标题和上映年份；
- 上次播放位置；
- 播放进度条；
- 海报或默认影视图标。

当前“继续播放”按钮保留为后续播放页迭代入口。

## 任务动态

读取最近四条统一后台任务，展示任务标题、状态、进度和最后更新时间。任务状态使用不同
颜色标识。

## 加载和失败处理

- 首次加载显示局部加载状态；
- 各数据接口独立容错；
- 单个接口失败不会阻止其他首页区域显示；
- 连接状态区域会提示不可用的数据项数量；
- 空数据库会显示明确的空状态，而不是错误信息。

本次迭代未新增后端接口、数据库字段或前端依赖。
