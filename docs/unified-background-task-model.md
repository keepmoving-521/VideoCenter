# Q01：统一后台任务模型

系统新增 `background_tasks` 作为跨业务后台任务的统一调度模型。当前下载、媒体扫描、视频
分析和 HLS 转码仍保留各自任务表；后续迭代可以逐类接入，不需要在本次升级中同时重构所有
已稳定运行的任务流程。

## 任务类型

- `download`：视频下载；
- `media_scan`：媒体目录扫描；
- `media_analysis`：视频信息分析；
- `hls_transcode`：HLS 转码；
- `generic`：其他通用后台任务。

## 统一状态

- `waiting`：等待执行；
- `running`：执行中；
- `paused`：已暂停；
- `completed`：已完成；
- `failed`：执行失败；
- `cancelled`：已取消。

状态集合覆盖现有四类任务，也为后续统一暂停、取消和恢复接口提供稳定契约。

## 公共字段

统一模型包括：

- 任务类型、状态、标题和优先级；
- `source_task_id`：关联迁移期间的旧业务任务，同一类型内唯一；
- `parent_task_id`：表达失败重试、派生任务等父子关系；
- 百分比进度、已处理数量和总数量；
- 当前尝试次数和最大尝试次数；
- 是否允许取消、是否支持暂停以及取消请求标记；
- 执行者编号和心跳时间；
- JSON 格式的任务参数和执行结果；
- 标准错误码、错误信息及完整时间字段。

数据库约束会保护进度、优先级、处理数量和重试次数范围。

## 接入边界

本次仅建立统一模型和读取数据结构，不新增任务 API，也不改变现有下载、扫描、分析和转码
接口。后续可按以下顺序继续迭代：

1. 建立统一任务创建与状态流转服务；
2. 将现有任务同步注册到统一表；
3. 提供统一任务列表、详情和控制接口；
4. 最终评估是否合并或保留业务任务明细表。
