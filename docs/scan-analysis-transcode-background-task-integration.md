# Q05～Q07：扫描、分析与转码接入统一任务

本地媒体扫描、视频信息分析和 HLS 转码已接入 `background_tasks` 统一任务中心。三类业务
任务仍使用原有表和执行器，统一任务负责汇总跨业务一致的状态、进度、结果和错误。

## 本地扫描任务

`scan_tasks` 创建后会注册 `media_scan` 类型统一任务，并同步：

- 扫描目录、关联影片和是否增量扫描；
- 等待、运行、完成和失败状态；
- 已处理文件数、总文件数和百分比进度；
- 新增、更新、跳过、丢失及恢复文件数量；
- 扫描错误和任务时间。

应用重启恢复等待或运行中的扫描任务时，会复用原统一任务并重置为等待状态。

## 媒体分析任务

`analysis_tasks` 会注册 `media_analysis` 类型统一任务，并同步：

- 待分析资源编号和强制分析选项；
- 已处理资源数、总资源数和百分比进度；
- 已分析、已跳过、缺失资源和逐项失败结果；
- 任务级失败信息。

分析重试会创建新的业务任务和统一任务，并通过 `parent_task_id` 指向原统一任务，形成可
追踪的重试链。

## HLS 转码任务

`hls_tasks` 会注册 `hls_transcode` 类型统一任务，并同步：

- 本地资源编号；
- 等待、运行、完成和失败状态；
- FFmpeg 实时转码进度与心跳；
- 输出目录和播放列表路径；
- FFmpeg 或文件相关错误。

重复请求复用已有 HLS 任务时，也会确保统一任务存在。应用重启恢复未完成转码时继续复用
原统一任务。

## 数据库

本次复用 Q01 已建立的统一任务类型，不修改数据库结构，不需要新增 Alembic 迁移。
