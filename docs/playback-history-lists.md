# H03～H04：继续观看与最近观看列表

系统提供面向首页和播放器的观看记录列表。列表不仅返回播放进度，还携带基础影片信息，
前端不需要再根据影片编号逐条查询标题和海报。

## 继续观看列表

```http
GET /api/v1/history/continue-watching?page=1&page_size=50
```

列表只包含：

- 播放位置大于 `0`；
- 总时长未知，或者播放位置小于总时长。

已经播放到总时长的影片和仅保存了 `0` 秒位置的影片不会进入继续观看列表。

## 最近观看列表

```http
GET /api/v1/history/recent?page=1&page_size=50
```

返回所有观看记录，并按照 `watched_at` 从新到旧排列。

## 分页与响应

两个接口都支持：

- `page`：页码，默认 `1`；
- `page_size`：每页数量，默认 `50`，最大 `200`。

响应包含 `items`、`total`、`total_pages`、`has_next` 和 `has_previous`。每条记录包含播放
位置、总时长、最近播放资源、更新时间，以及影片的标题、类型、上映年份和海报地址。

## 数据库

本次迭代复用现有 `watch_history` 和 `media` 表，没有修改数据库结构，不需要新增
Alembic 迁移。
