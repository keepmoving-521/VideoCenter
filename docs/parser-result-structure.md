# R02：解析结果数据结构

R02 将 R01 中的最小结果对象升级为经过校验、可序列化的标准数据模型。具体网站解析器必须返回这些结构，避免各解析器使用不同的字典字段。

## 层级结构

```text
ParseResult
├── 影片元数据
├── downloads[]：影片级下载资源
└── seasons[]
    └── episodes[]
        └── downloads[]：分集下载资源
```

## 主要类型

| 类型 | 说明 |
| --- | --- |
| `ParsedMediaType` | 电影、电视剧、纪录片、动画、综艺、短片和其他 |
| `ParsedResourceType` | 视频、音频、字幕和其他资源 |
| `ParsedDownload` | 地址、清晰度、语言、文件大小、请求头等下载信息 |
| `ParsedEpisode` | 集号、标题、播出日期、时长、缩略图和下载资源 |
| `ParsedSeason` | 季号、标题、播出日期、海报和分集 |
| `ParseResult` | 完整影片元数据、季集结构和扩展数据 |

## 校验规则

- 页面、图片和下载地址仅接受 HTTP/HTTPS。
- 影片标题、来源网站和来源页面地址必填。
- 影片评分范围为 0～10。
- 时长和文件大小必须大于 0。
- 季号从 0 开始，集号从 1 开始。
- 同一影片不能出现重复季号，同一季不能出现重复集号。
- 只有 `series` 类型可以包含季。
- 别名、演职员、地区、语言、类别和标签会去除空白并按大小写去重。
- 模型不可变，禁止写入未定义字段。

## 序列化

解析结果可转换为适合 API 或任务存储的 JSON 数据：

```python
payload = result.model_dump(mode="json")
```

日期和枚举会自动转换为字符串。只需要创建 `Media` 的字段时，可使用：

```python
media_values = result.to_media_values()
```

该方法会排除标签、下载资源、季集和解析器私有扩展数据，后续迭代可分别处理这些关联对象。

本次迭代不修改数据库结构，不需要新增 Alembic 迁移。
