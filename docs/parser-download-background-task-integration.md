# Q03～Q04：资源解析与视频下载接入统一任务

资源解析和视频下载已接入 `background_tasks` 统一任务中心。原有业务接口和任务表继续保留，
统一任务负责提供跨业务一致的状态、进度、错误、结果和时间信息。

## 资源解析任务

调用预解析接口时，系统会：

1. 创建 `resource_parse` 类型的统一任务；
2. 将状态从 `waiting` 更新为 `running`；
3. 保存原有 32 位 `parse_task_id`、页面地址和首选语言；
4. 解析成功后记录标题、来源网站以及检测到的下载和字幕数量；
5. 解析失败时保存标准错误码和错误信息。

预解析响应新增：

```json
{
  "parse_task_id": "原有日志任务编号",
  "background_task_id": 42
}
```

原有 `parse_task_id` 保持不变，用于兼容解析日志、重复解析检测和现有客户端。

## 视频下载任务

创建下载任务时同步创建 `download` 类型统一任务，并通过 `source_task_id` 关联原
`download_tasks.id`。同一下载任务重复同步不会生成重复统一任务。

以下变化都会同步：

- 等待、下载中、暂停、完成、失败和取消状态；
- 优先级与下载百分比；
- 已下载字节数和总字节数；
- 首次开始时间、心跳时间和结束时间；
- 下载错误；
- 完成后的文件路径、文件大小和 SHA-256。

失败任务重新排队时，统一任务的 `attempt` 自动增加。

## 兼容边界

- 下载队列仍以 `download_tasks` 为业务数据源；
- 解析预览结果仍由现有临时工作流存储管理；
- 本次不新增统一任务查询或控制 API；
- 统一任务表是状态汇总中心，不取代业务专属字段。
