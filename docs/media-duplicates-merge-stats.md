# L14～L16：重复影视检测、资源合并与影视库统计

## 功能范围

- L14：检测影视库中的疑似重复条目。
- L15：将确认重复的条目合并到用户指定的目标影视。
- L16：统计影视库、收藏、本地资源、下载任务、标签和季集数量。

本次迭代只增加业务接口，不改变数据库结构，因此不需要新增 Alembic 迁移。

## 重复检测规则

`GET /api/v1/media/duplicates`

满足任意规则的条目会进入同一重复组：

1. 原始资源页面地址相同。比较时忽略协议和域名大小写、URL 片段及末尾斜杠。
2. 规范化标题、上映年份和影片类型同时相同。标题比较会忽略大小写、空格和标点。

只有标题但缺少上映年份的条目不会按标题规则自动判重，以减少误报。响应中的
`reasons` 会说明每个重复组命中了哪些规则。

## 重复资源合并

`POST /api/v1/media/merge`

请求示例：

```json
{
  "target_media_id": 1,
  "source_media_ids": [2, 3]
}
```

合并行为：

- 保留 `target_media_id` 指定的影视条目。
- 目标条目缺少的元数据会从来源条目中补齐。
- 列表型元数据、标签会去重合并。
- 收藏状态只要任意条目已收藏，合并后即保持收藏。
- 本地资源和下载任务改为关联目标条目，不删除磁盘文件。
- 相同季号、集号的数据会合并；不同季或集会迁移到目标条目。
- 观看历史保留 `watched_at` 最新的一条。
- 完成迁移后删除来源影视条目。

接口只允许合并当前规则检测出的重复条目。非重复条目返回
`MEDIA_NOT_DUPLICATE`，条目不存在返回 `MEDIA_NOT_FOUND`。

## 影视库数量统计

`GET /api/v1/media/stats`

响应包含：

- 影视总数和收藏数。
- 已关联本地资源的影视数。
- 本地资源、下载任务、标签、季和分集总数。
- 按影片类型和资源状态分组的数量。
