# L09-L11：影视信息编辑、删除与批量删除

## 编辑影视信息

```http
PATCH /api/v1/media/{media_id}
```

接口支持更新影视元数据、收藏状态、个人评分和备注，并返回包含标签、本地资源、季和分集的完整详情。

已经存在季数据的电视剧不能直接修改为其他影片类型，错误代码为 `MEDIA_TYPE_HAS_SEASONS`。应先删除季数据，再修改影片类型。

## 删除单个影视资源

```http
DELETE /api/v1/media/{media_id}
```

删除规则：

- 删除影视条目；
- 删除该影片的观看历史；
- 删除季和分集；
- 删除影片与标签的关联，但保留标签；
- 本地资源记录保留并解除 `media_id`；
- 下载任务保留并解除 `media_id`；
- 不删除磁盘上的实际视频文件。

不存在的影视资源返回 `MEDIA_NOT_FOUND`。

## 批量删除

```http
POST /api/v1/media/batch-delete
```

请求体：

```json
{
  "media_ids": [1, 2, 3]
}
```

每次允许提交 1～100 个正整数 ID，重复 ID 会自动去重。响应：

```json
{
  "deleted_count": 2,
  "deleted_ids": [1, 2],
  "missing_ids": [3]
}
```

批量删除采用与单个删除相同的关联清理规则，并在一个数据库事务中完成。

本次迭代只调整 ORM 关联行为和业务删除逻辑，不修改数据库结构，不需要新增 Alembic 迁移。
